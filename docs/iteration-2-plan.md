# План Ітерації 2 (Lab 35)

## 1. Сценарії з Backlog
У цій ітерації реалізуються наступні Use Cases:
1. Збереження та відновлення списку задач із JSON-файлу (асинхронно).
2. Фільтрація та аналітика задач (LINQ-запити: за статусом, за пріоритетом, підрахунок статистики).
3. Зміна статусу задачі з перевіркою бізнес-правил та застосуванням патерну.

## 2. Класи, що залишаються без змін
Базові сутності `BaseEntity`, `User`, `TaskStatus`, `TaskPriority` з Lab 34 залишаються практично без змін, оскільки вони спроєктовані з урахуванням інкапсуляції.

## 3. Точки розширення (Патерни)
Буде використано патерн **Strategy (Стратегія)** для гнучкого оцінювання терміновості задачі (UrgencyStrategy). Це дозволить легко додавати нові правила розрахунку пріоритетів у майбутньому без зміни класу `TaskItem`.

## 4. Ризики
- **Серіалізація закритих властивостей:** `System.Text.Json` може мати проблеми з десеріалізацією класів, де `set` є `private`. Вирішення: використання атрибутів `[JsonInclude]` або налаштування опцій серіалізатора.
- **Слабкі інтерфейси:** Поточний `ITaskRepository` синхронний. Доведеться додати асинхронні методи `SaveAsync` та `LoadAsync`.
