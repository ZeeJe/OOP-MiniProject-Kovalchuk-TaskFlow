# Test Strategy

## Критичні сценарії
1. Розрахунок терміновості (Strategy).
2. Валідація статусів (неможливість повернути завершену задачу в ToDo).
3. Відновлення даних із JSON-файлу (інтеграція).

## Тестування залежностей
Для `TaskService` використовується фреймворк `Moq`, що дозволяє ізолювати бізнес-логіку від реального I/O і перевіряти методи швидко й автономно.

## Fault Handling (Негативні сценарії)
Файловий репозиторій має вбудований захист (try-catch на `JsonException`). Якщо файл пошкоджений, система не падає (crash), а ініціалізує порожню базу, дозволяючи користувачеві продовжити роботу.
