# Vision: TaskFlow (Система управління задачами)

## Короткий опис
TaskFlow — це застосунок для організації особистих та командних завдань. Дозволяє створювати задачі, призначати виконавців, групувати їх за проєктами та відслідковувати статус виконання.

## Цільові користувачі
- Індивідуальні користувачі (фрілансери, студенти) для трекінгу власних справ.
- Невеликі команди для розподілу завдань у проєктах.

## Основні сценарії використання (Use Cases)
1. Створення нової задачі (з вказанням пріоритету та опису).
2. Зміна статусу задачі (To Do -> In Progress -> Done).
3. Перегляд списку всіх активних задач проєкту.

## Нефункціональні вимоги
- **Тестованість:** Бізнес-логіка повинна бути повністю покрита unit-тестами.
- **Масштабованість:** Архітектура повинна дозволяти легку заміну In-Memory сховища на реальну БД (напр., SQLite або MySQL).
- **Ізоляція шарів:** UI не має прямого доступу до бази даних (тільки через Application шар).

## Обмеження першої ітерації (Lab 34)
Свідомо НЕ реалізується: постійне зберігання у файл/БД (використовується In-Memory), багатокористувацька авторизація, складні фільтри задач. Фокус на доменній моделі та одному вертикальному зрізі.
